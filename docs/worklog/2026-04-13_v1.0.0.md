오늘은 약먹자 앱을 일단락 짓고 깃허브에 처음으로 올리는 작업이 중심이었다. 새벽부터 복용량과 복용 시간 입력 UX 를 정리하고, 자잘한 버그를 계속 잡다가 릴리즈 빌드까지 뽑은 뒤 초기 커밋으로 원격에 푸시했다. 그 사이에 메모요 앱은 가벼운 디자인 보정만 몇 건 돌아갔고, 저녁부터는 텔레그램과 Claude Code 채널스를 연동하는 방향에 대해 긴 리서치와 상담이 이어졌다. 실제 코드 작업량은 앞 이틀보다 적었지만 환경과 도구를 정돈한 날이었다.

새벽 시간대의 약먹자 작업은 복용량 입력 UI 를 복용 시간 입력과 동일한 모양으로 맞추는 데서 시작했다. 복용량 네모 카드 안에 복용량 라벨과 추가 버튼을 복용 시간처럼 배치하고, 추가 버튼을 누르면 팝업이 뜨며 입력할 수 있게 만들었다. 입력 후 복용량이 가운데 정렬로 찍히던 것이 어색해 복용 시간과 마찬가지로 왼쪽 정렬로 바꿨다. 시간 선택 휠의 오전 0 시와 오후 12 시 표기가 직관적이지 않다는 지적이 있어 서브 에이전트들과 토론을 거쳐 중복 없는 24 시간 관점의 표기로 정리했다. 복용 시간 칸 아래에는 다음 복용까지 남은 시간을 실시간 카운트다운으로 표시하는 영역을 붙였고, 복용 체크 후 카운트다운이 사라지면서 레이아웃이 살짝 튀던 문제는 고정 높이를 잡아 해결했다. 같은 약에 같은 시간을 두 번 등록하려 하면 중복 방지 로직으로 막았고, 등록된 약이 하나도 없을 때 화면이 스크롤로 흔들리던 문제는 반응형으로 고쳐 내용이 있을 때만 스크롤되게 만들었다.

기능이 어느 정도 자리 잡은 뒤에는 릴리즈 빌드로 넘어갔다. 빌드 클린과 릴리즈 런을 한 줄로 이어서 실행하는 명령을 정리해 CLAUDE.md 에 박아 두기로 했다. 이 규칙이 모든 프로젝트에서 공유돼야 한다는 이야기가 나와 프로젝트 로컬 CLAUDE.md 가 아닌 공용 설정 쪽에 넣는 방향으로 정리했다. 그 밖에 기능 대비 용량이 크다는 피드백이 있어 APK 최적화도 한 번 돌려 크기를 줄이고, 아이폰 빌드 명령도 세트로 정리해 기록했다. 이 모든 변경이 어느 정도 모여 약먹자 프로젝트를 깃허브에 처음 올릴 준비가 됐고, 기존 시도 흔적을 지운 깨끗한 상태로 4981029 Initial commit 약먹자 약 복용 알림 Flutter 앱 커밋이 만들어져 원격 저장소에 푸시됐다. 이 커밋이 약먹자 저장소의 공식 시작점이다. 작업 말미에는 단순 대화도 많이 섞였고, 새벽 한 시 무렵 사용자가 자러 간 뒤 자리를 정리했다.

메모요 쪽 작업은 분량이 많지 않았다. 워크트리 환경에서 메모요 앱의 아이폰 빌드 명령을 클린과 릴리즈 런 한 줄 형태로 정리했고, 메모 선택 시 상하 여백이 아래쪽으로 치우쳐 가운데 정렬이 무너지던 문제를 수정했다. 메모 사이에 들어 있던 노란색 구분선 요소가 좌우 여백 때문에 화면 끝까지 채워지지 않던 것도 패딩을 걷어 전체 너비로 늘려 정리했다. 이 수정들은 워크트리 안에서만 이어졌고 본 브랜치 푸시는 없었다.

저녁부터는 하루의 성격이 리서치로 넘어갔다. 텔레그램이나 디스코드로 페이크 채팅 같은 UI 를 연동할 수 있는지에서 시작해 Claude Code Channels 라는 기능이 현재 프리뷰 단계라는 점, 로컬 맥에서 실행되며 Claude Max 요금제 사용량과 같은 풀을 쓴다는 점, 리서치 프리뷰 이후 유료 전환 가능성 같은 부분을 하나씩 정리했다. 이미 클로드 데스크탑 앱의 디스패치 기능과 비교했을 때 어떤 장점과 단점이 있는지도 같이 비교했다. 디스패치는 폰에서 명령을 내리면 내 맥의 Claude Code 가 실제 작업을 돌리는 구조고, 채널스는 슬랙이나 텔레그램 같은 채팅 계층을 가운데에 두어 비동기 알림과 대화 인터페이스를 제공하는 방향이라는 식으로 정리했다. 결론은 텔레그램을 채널로 붙여 사이드 프로젝트 브리핑 흐름을 만들고 싶다는 쪽으로 기울었고, 채널스 설정은 Claude Code 세션이 아니라 별도 채팅 세션에서 진행하기로 미뤘다.

그 밖에 잡다한 작업도 있었다. 로그인 메서드와 모델 설정을 잠깐 점검하면서 claude login 과 claude config set model opus 명령으로 구성을 다시 확인했고, 세션 하나가 소넷으로 잡혀 있어 왜 소넷이냐는 질문이 나왔다. fvm 관련 빌드 명령, 레포지토리 개념, git push 와 커밋의 차이 같은 기초 개념 상담도 중간에 짧게 섞였다.
