2026.04.11 작업일지 v1.0.0

오늘은 메모 앱 메모요를 처음부터 만들기 시작해서 기본 기능과 외형을 거의 다 잡은 날이었다. 하루 전체가 simple_memo_app 저장소 하나에 집중됐고, 중간에 개인 홈페이지 daejong-page 초판까지 함께 올렸다. 코드 한 줄부터 APK 빌드, 아이콘 제작, 그리고 iOS 재실행 크래시 디버깅까지 프로젝트 라이프사이클의 주요 단계가 하루에 압축돼 지나갔다.

Flutter 로 메모 앱 뼈대를 처음 잡을 때는 단순하게 시작했다. 제목 없이 본문만 입력하는 형식을 원해서 입력란을 하나로 줄였고, 목록에는 본문 첫 줄만 보이도록 정리했다. 로컬 저장은 SharedPreferences 를 쓰고 모든 UI 는 한글로 작업했다. 다크 테마를 기본으로 두고 + 버튼과 폰트 색상을 톤에 맞춰 다듬었다.

오후부터는 제스처 부분을 붙잡고 계속 이터레이션 했다. 처음에는 오른쪽으로 스와이프하면 빨간 삭제 버튼, 왼쪽으로 하면 노란 별 버튼으로 즐겨찾기가 등록되는 방식으로 시작했는데, 이게 아이폰 메모앱과 방향이 반대라 금방 바꿨다. 왼쪽 스와이프에 삭제, 오른쪽 스와이프에 즐겨찾기를 배치하고, 삭제 버튼 색은 빨강 대신 다른 톤으로 바꿨다. 스와이프한 뒤 버튼이 그대로 고정되어 있다가 버튼을 직접 누르거나 완전히 바깥을 탭하면 닫히도록 만들었다. 이 과정에서 살짝만 움직여도 버튼이 닫히는 버그, 다른 곳 탭해도 안 닫히는 버그, 버튼이 열려 있는데 클릭이 안 먹는 버그 같은 것들을 연달아 잡았다.

드래그로 메모 순서를 바꾸는 기능을 넣는 건 생각보다 오래 걸렸다. 즐겨찾기와 일반 메모를 한 화면에 보여주는 건 쉬웠는데, 드래그 도중 한 그룹의 메모가 다른 그룹 영역으로 넘나들며 섞이는 문제가 반복됐다. 같은 그룹 안에서만 인덱스를 계산하도록 여러 번 고쳐도 드래그 프리뷰에서 경계가 무너지는 현상이 남아서, 결국 기존 구현을 버리고 ReorderableListView 를 두 개로 완전히 분리하는 방향으로 다시 만들었다. 즐겨찾기 그룹과 일반 그룹을 독립된 리스트로 둔 뒤부터는 드래그 중에도 서로 영향을 주지 않고 각 리스트가 자기 안에서만 움직이게 돼 안정적으로 동작했다. 그룹 사이에 있던 구분선도 지웠다.

앱 이름과 아이콘 작업도 병행했다. 다운로드 폴더에 있던 메모요 SVG 아이콘을 기반으로 APK 앱 아이콘을 교체했는데, 처음 빌드해보니 화면에서 아이콘이 너무 작게 보여서 여러 번 다시 뽑았다. 검은 배경을 없애고 고양이와 노트만 더 크게, 노트 줄은 검게 더 굵게 고치고, 배경은 투명하게 만드는 식으로 반복했다. 아이콘 이미지는 중간중간 파일로 직접 열어 확인했다. 앱 이름은 영어로 잡혀 있던 걸 한글 메모요로 바꿨다. APK 빌드를 위해 android-commandlinetools 를 설치하고 툴체인을 정비했고, 용량 최적화까지 한 번 돌려서 결과물을 가볍게 만들었다.

메모요 우측 상단에는 카카오페이 응원 버튼을 붙였다. 누르면 qr.kakaopay.com 의 내 후원 페이지가 브라우저에서 열리게 했고, 다크 테마와 어울리도록 작고 세련된 톤으로 배치했다. 처음엔 후원하기 라는 이름이었는데 오후 늦게 응원하기 로 바꾸고, 하트 아이콘 위치도 8픽셀 왼쪽으로 옮겨 미세하게 다듬었다. 즐겨찾기 별 아이콘을 직접 탭하면 즐겨찾기가 바로 해제되게 하는 동작도 이 시점에 추가했다.

저녁에는 처음 만든 개인 홈페이지 daejong-page 를 올렸다. daejong-page 폴더에 index.html 을 새로 만들고, 다크모드 개발자 소개 페이지로 디자인했다. 이름 강대종, 소개 문구, 카카오페이 응원하기 버튼을 배치했고, 이 페이지를 깃허브 페이지로 띄워 메모요 앱의 응원하기 버튼이 여기로 연결되도록 이어 붙였다. 홈페이지 자체의 저장소도 이날 초기 커밋으로 생성됐다.

밤 늦게 사용자 제보가 들어왔다. 첫 실행은 정상인데 앱을 종료했다 다시 열면 곧바로 크래시가 난다는 내용이었다. iOS 에서 재실행 시 무조건 터지는 패턴이라 원인을 lib 전반에서 찾기 시작했다. main.dart 부터 읽어 내려가면서 MemoStorage 의 SharedPreferences 로드, jsonDecode, late 변수와 ! 연산자, initState 비동기 처리, null 처리가 빠진 지점을 전수 조사했다. 문제 후보를 좁혀 가며 try-catch 와 null 가드를 보강하기 시작했지만 완전한 해결은 다음 날로 넘어갔다.

그 밖에 개발 환경을 다듬는 작은 작업도 있었다. 터미널 Ghostty 에 JetBrains Mono 폰트를 설치하고 config 를 만들어 붙여, 다음 작업 때 가독성이 좀 나아지도록 정리해 뒀다. 깃 원격 설정이 뭔지, 커밋이 뭔지 같은 기본 용어에 대한 짧은 대화도 오갔다. 이 자리에서 simple_memo_app 의 첫 커밋과 daejong-page 의 첫 커밋이 각각 만들어졌다.

남은 작업은 재실행 크래시의 근본 원인을 완전히 잡는 것과, 즐겨찾기 드래그의 세부 이슈들이 여전히 남아 있는지 실기기에서 한 번 더 검증하는 일이다.
