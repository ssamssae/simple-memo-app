2026.04.19 작업일지 v1.0.4

v1.0.3 까지가 오후 할일 자동화 안정화와 더치페이 계산기 1.0.2+4 정식화, 그리고 APK 용량 최적화로 마무리된 지점이었다면, v1.0.4 는 그 뒤에 이어진 야간 러너 파이프라인 구축과 첫 실전 PR 2건 생성까지의 흐름을 담는다.

저녁 7시 전후로 WSL 데스크탑의 night-runner 가 실제로 첫 결과물을 밀어 올렸다. dutch_pay_calculator 에는 janitor 2026-04-19 브랜치로 다섯 커밋이 올라왔고, simple_memo_app 쪽에도 같은 브랜치명으로 네 커밋이 올라왔다. 더치페이 쪽은 기본 Flutter 템플릿이었던 README 를 실제 앱 설명으로 교체하고, widget_test.dart 를 카운터 샘플에서 splash timer leak 수정 더하기 계산기 위젯 흐름 커버 테스트로 확장했고, pubspec description 도 문구를 정돈했으며 pubspec.lock 의 vm_service 를 15.0.2 에서 15.1.0 으로 올린 뒤 session 로그까지 남긴 구조다. 메모요 쪽도 기본 카운터 위젯 테스트를 Memo 모델 단위 테스트로 교체하고, memo_list_screen 의 reorder 가드 if 블록에 중괄호를 보강하고, README 의 프로젝트 구조 섹션을 현재 lib 트리 기준으로 갱신한 다음 janitor 로그를 붙이는 방식이었다. 두 쪽 다 conventional commits 규칙에 따라 docs style test chore 같은 접두사로 atomic 하게 쪼개져 있어 품질이 괜찮았다.

이 결과물을 Mac 본진에서 어떻게 받을지가 그 다음 과제였다. 데스크탑에서 이미 morning-briefing 의 1-E 섹션을 추가해 janitor PR 을 아침 7시 15분 통합 브리핑에 끼워 넣는 쪽은 밀어 올려 둔 상태였지만, 그 PR 에 대해 사용자가 텔레그램 답장으로 PR 3번 머지 같은 문장을 쳤을 때 실제로 gh pr merge 가 돌아가는 핸들러는 비어 있었다. 이를 풀기 위해 /merge-janitor 라는 새 스킬을 claude-skills repo 에 작성했다. 구조는 1단계에서 gh search 로 open 상태 janitor PR 을 번호로 찾고, 2단계에서 gh pr view 로 diff 메타데이터를 가져와 변경 줄 수 300 초과 여부와 pubspec Podfile Gemfile 같은 의존성 파일 포함 여부와 .env key token secret pem p12 keystore 같은 민감 파일 포함 여부와 CI 키스토어 변경 여부를 체크한 뒤 3단계에서 경고가 있으면 사용자 확인을 기다리고 경고가 없으면 곧바로 gh pr merge 를 실행하는 흐름이다. 닫기 모드는 gh pr close --delete-branch 한 줄로 분기했다. 처음엔 gh search 한 번에 additions deletions files 까지 받아오려다가 그 필드들은 search 에서는 지원이 안 된다는 걸 검증 중에 발견해 2단계로 분리했다.

머지 전략 기본값은 squash 로 시작했다가 대종님 지적으로 다시 뒤집었다. janitor 커밋이 이미 atomic 하게 잘 쪼개진 conventional commits 인데다 개인 repo 의 git log 가 그 자체로 기억 보조장치라, squash 하면 bisect 와 blame 해상도가 떨어진다는 판단이었다. 그래서 default 는 --merge 로 바꾸고, 필요할 때만 /merge-janitor N squash 나 --squash 플래그를 받아 squash 모드로 들어가도록 옵션화했다. 나중에 WIP 반복 커밋 패턴이 쌓이는 프로젝트가 생기면 그 때 기본값을 커밋 수 휴리스틱으로 전환하자는 방향성도 스킬 본문에 적어 뒀다. 자연어 트리거는 skill description 안에 PR N 머지와 PR N 닫아 같은 패턴을 적어 두어 Mac 과 WSL 양쪽에서 claude-skills pull 만 받으면 바로 인식되게 했고, 맥 본진의 CLAUDE.md 에도 같은 문장을 한 줄 추가해 보강했다.

파이프라인이 완성된 뒤 첫 실전 PR 두 개를 바로 만들었다. 더치페이 쪽은 맥이 직접 gh pr create 로 PR 1번을 올렸고, 제목은 janitor 2026-04-19 야간 잡일 처리, 본문에 다섯 커밋 목록과 변경 규모와 안전 체크 결과를 담았다. 메모요 쪽은 WSL 의 gh auth 가 아직 안 돼 있어서 대종님이 브라우저로 직접 compare 페이지에서 Create pull request 를 눌러 PR 1번을 올렸다. 맥에서 gh pr list 로 두 PR 이 전부 open 상태로 잡히는 것을 확인했고, 이로써 내일 07시 15분 morning-briefing 의 야간 러너 PR 섹션이 첫 실전 입력을 받게 된다.
