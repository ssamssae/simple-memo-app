2026.04.15 작업일지 v1.0.0

오늘은 세션을 열 때부터 텔레그램 채널로 대화하면서 개발 환경 전반을 다듬고 개인 홈페이지를 본격적으로 키우는 하루였다. 단일 앱 작업보다는 도구와 스킬, 공개 페이지에 초점을 둔 날이었다.

더치페이 계산기 쪽은 한 가지만 잡았다. 앱 아이콘 네 귀퉁이가 흰색으로 남아 있던 문제를 파이썬에서 플러드 필로 노란색 FFB300 으로 채우고, 15개 사이즈 모두 재생성했다. 투명 채널도 함께 제거해서 앱스토어 심사에서 걸리지 않게 마무리했다.

도구 쪽은 worklog 스킬을 크게 키웠다. 기존에는 현재 프로젝트 하나만 기록하던 것을 고쳐, 지정 날짜에 활동이 있었던 모든 클로드 코드 세션 jsonl 을 스캔해 유저 질문과 응답, 툴 사용 내역을 모아 요약하도록 바꿨다. 파일명도 앱스토어 버전처럼 YYYY-MM-DD_vX.Y.Z.md 형식으로 바꿔, 같은 날짜에 여러 번 호출해도 이전 파일은 보존되고 패치 버전이 하나씩 올라가도록 했다. 동시에 daejong-page 공개 홈페이지의 worklog 폴더와 index.json 에도 자동으로 복사·커밋·푸시되는 단계도 붙였다.

todo 스킬과 미리알림 연동도 새로 만들었다. 홈 폴더에 todo 저장소를 만들고 todos.md 를 진행중, 완료, 보류, 사이드 프로젝트 아이디어 네 섹션으로 나눴다. 맥OS 미리알림 앱에 Claude 라는 새 목록을 만들고 AppleScript 로 추가·완료·삭제 흐름을 검증했다. CLAUDE.md 에는 자연어 대화만으로도 할일 등록과 완료 처리가 되도록 자동 규칙을 박아 두었고, 첫 번째 할일로 개인 홈페이지 버전업을 등록했다.

그 개인 홈페이지 daejong-page 는 v1.0.0 스냅샷을 만드는 데서 시작해 v1.0.1 에서 ssamssae 네이버 이메일 버튼을 추가하고, v1.1.0 에서 작업일지 페이지와 worklog 폴더, 홈 카드 링크를 붙였다. v1.1.1 에서는 작업일지 링크를 정식 고스트 버튼으로 승격해 3버튼 레이아웃으로 정리했고, v1.2.0 에서는 소개문 아래에 since 2026.04.11 표기와 포트폴리오 페이지 뼈대를 올려 4버튼 구성으로 마무리했다. 각 버전마다 v디렉터리 스냅샷과 git 태그를 함께 남겨 과거 버전도 라이브로 접근 가능한 상태다.

어제 날짜 작업일지도 이번 체계에 맞춰 새로 썼다. 메모요 앱 리디자인과 배포, 그리고 텔레그램 봇 연동 여정을 한 문서로 통합한 2026-04-14_v1.0.0 버전을 simple_memo_app 저장소에 올렸다. 기존에 앱 작업만 담겨 있던 레거시 파일은 그대로 두고 별도 버전으로 구분했다.

하네스 권한도 손봤다. Edit, Write, Read, Glob, Grep, Skill, Bash 를 전체 허용으로 올려 반복 팝업을 줄였고, 자동 진행 규칙이 저장된 자동 메모리와 역할이 겹치지 않도록 정리했다. 자동 메모리에는 개인 홈페이지 관련 정보도 새로 한 건 추가해서 다음 세션에서도 맥락이 이어지도록 했다.

그 밖에 텔레그램 채널로는 여러 상담성 대화도 함께 흘러갔다. 무료 대안 AI 인 제미나이와 그록을 활용한 텔레그램 봇 수익화 아이디어, 크몽 외주 방향, 맥 파워 냅이 깨어나는 이유, 깃허브 페이지 무료 한도, 한 도메인으로 서브도메인과 경로를 이용해 여러 사이드프로젝트를 운영하는 법 같은 것들을 정리했다.

남은 작업은 포트폴리오 카드 내용을 실제 앱 정보로 채우는 것과, 메모요 앱 응원 버튼이 열어 주는 URL 이 계속 정상 경로를 가리키는지 다음 빌드에서 한 번 확인하는 정도다.
