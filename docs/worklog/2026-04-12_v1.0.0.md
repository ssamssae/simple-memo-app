오늘은 하루 종일 여러 앱을 오갔다. 메모요의 iOS 재실행 크래시를 본격적으로 잡고, 더치페이 계산기 앱을 처음부터 만들고, 약먹자 앱을 거의 재설계 수준으로 뜯어 고쳤으며, 고독사 방지앱이라는 새 프로젝트도 초기 세팅까지 올렸다. 한 프로젝트에 한 시간씩 머물다 넘어가는 식이 아니라 앱 네 개가 거의 병렬로 돌아간 날이었다.

메모요 쪽은 전날 밤에 남겨둔 재실행 크래시 디버깅에서 출발했다. iOS 에서 앱을 껐다 켜면 즉시 EXC_BAD_ACCESS 로 죽는 증상이었고, 원인은 결국 Flutter 3.38 부터 필수가 된 iOS UIScene 라이프사이클 마이그레이션 누락이었다. FlutterImplicitEngineDelegate 와 UIApplicationSceneManifest 를 쓰는 조합에서 두 번째 실행 때 Scene 복원이 엔진 초기화보다 먼저 일어나면서 SharedPreferencesPlugin 이 등록 도중 터지는 경로였다. AppDelegate, Info.plist, Podfile 을 정렬해 Scene 설정을 정리하고, UIScene 자체를 제거하는 방향으로 돌렸다. 방어 차원에서 runZonedGuarded 로 Dart 레벨 에러 캐치를 붙이고, decodeList 가 손상 데이터를 만나도 무너지지 않도록 try-catch 와 기본값 반환을 넣었다. 동시에 메모 ID 를 millisecondsSinceEpoch 에서 UUID v4 로 바꿔 같은 밀리초 내 충돌 가능성을 제거했고, 편집 모드에서 빈 내용으로 뒤로 가면 삭제 확인 다이얼로그가 뜨도록 UX 를 보강했다. Reorder 콜백에는 인덱스 범위 초과 방어 가드를, 스와이프 드래그에는 AnimatedContainer duration zero 처리를 넣어 지연을 없앴다. 마지막으로 deprecated withOpacity 호출을 withValues(alpha:) 로 전환해 경고도 정리했다. 이 일련의 수정은 2c467cc 부터 2f9f39e, 1318ea0, 88e6c2e, 19aecd6 등 여러 커밋으로 저장됐고 원격에 모두 반영됐다.

메모요 디자인 쪽에서는 한 화면에 메모가 최소 11개 보이도록 컴팩트 리스트 스타일로 바꾸는 작업을 했다. 즐겨찾기 섹션과 일반 섹션의 제목 텍스트를 없애고, 여백을 iOS 기본 메모 앱처럼 빡빡하게 줄였다. 드래그 핸들 아이콘이 오른쪽 스크롤 휠과 겹치던 문제는 trailing 쪽 패딩을 더해 왼쪽으로 살짝 밀었다. AppBar 의 메모장 타이틀은 안드로이드에서 왼쪽 정렬되던 걸 centerTitle 로 양쪽 플랫폼 정렬을 통일했고, 메모 편집 화면의 메모 수정과 새 메모 타이틀도 같은 방식으로 가운데 정렬했다. 앱 실행 첫 화면으로는 1.5 초짜리 흰 배경 스플래시를 새로 넣었다가, 체감이 길다는 피드백에 따라 곧바로 750 밀리초로 반으로 줄였다. APK 도 split-per-abi 로 분리 빌드해 아키텍처별 용량을 줄였고, iOS 아이콘의 알파 채널을 모두 제거해 App Store 업로드 거부를 미리 막았다. 안드로이드 빌드를 위해서는 Android Studio 와 Android SDK 환경변수까지 새로 설치하고 경로를 잡았다.

메모요의 Dart 레벨 방어 로직이 어느 정도 완성된 시점에서 AGENTS.md 작업 규칙 파일을 프로젝트 루트에 새로 추가했다. 모든 코드 수정이 끝난 뒤에는 반드시 git add, commit, push 까지 자동으로 돌리고, 커밋 메시지는 한글로 fix 또는 feat 접두어를 쓴다는 규칙을 명시해 두었다.

더치페이 계산기 앱은 오늘 아예 새로 만들었다. dutch_pay_calculator 프로젝트를 생성하고 번들 ID 를 com.daejongkang.dutchPayCalculator 로 잡은 뒤, 스플래시 1.5 초와 앱 아이콘, 한국어 앱 이름 더치페이 계산기 까지 한 번에 세팅했다. 초기 UI 는 금액과 인원수를 입력하면 1 인당 금액이 계산되는 단순 구조로 시작했는데, 사용자 피드백을 받아 금액 입력 영역을 토스 송금 화면 스타일로 다시 만들었다. 시스템 키보드를 아예 쓰지 않고 화면 하단에 커스텀 숫자 키패드를 직접 그려 붙였고, 키는 4x3 그리드로 정렬했다. 처음엔 00 자리에 C 를 넣었다가 배치가 어색해 00 을 없애고 C, 0, ⌫ 세 개만 남기는 방향으로 정리했다. 인원수 조절은 가운데 정렬의 −, 숫자, + 버튼으로 바꾸고 인원수 라벨은 박스 바깥 아래로 뺐다. 금액 텍스트가 입력 상태와 빈 상태에서 높이가 달라 UI 가 위아래로 흔들리던 버그는 두 상태의 높이를 같게 고정해 잡았다. 작은 안드로이드 폰에서 세로로 스크롤이 생기던 문제는 MediaQuery 기반 반응형 사이즈로 바꿔 한 화면에 꽉 차게 만들었다. 40.5 메가바이트까지 올라갔던 APK 용량은 안 쓰는 패키지 제거, 에셋 최적화, split-debug-info 와 obfuscate 옵션, 불필요한 아키텍처 제거를 동시에 적용해 많이 줄였다. App Store Connect 업로드를 준비하다 iOS 아이콘 알파 채널 때문에 거부당한 것도 이날 바로 잡았다. 메모요와 같은 스타일의 응원 버튼도 오른쪽 상단에 달았고, 링크는 daejong-page 홈으로 이어지게 했다.

약먹자 앱은 저녁부터 완전히 갈아엎었다. 기존 코드는 다른 세션에서 엉망으로 시작된 상태였고, 일단 현재 상태를 초기 커밋으로 깃허브에 올린 뒤 재설계 방향으로 다시 만들기 시작했다. 아이콘은 노란색과 흰색의 알약 모티브로 바꾸고, 로딩 화면의 푸른 톤도 앱 아이콘과 맞춰 정리했다. 배경을 전면 노란색으로 두는 게 부담스러워 다크톤으로 전환했고, 앱 전체를 다크모드 기조로 통일했다. 약 복용량 입력은 0.5 알부터 10 알까지 스크롤 휠로 돌려 선택하는 방식으로 바꾸고, 복용 시간 세 개도 같은 방식으로 통일해 기존의 시계 다이얼 입력을 제거했다. 저장 버튼은 키보드가 올라왔을 때 가려지지 않도록 우측 상단 AppBar 로 옮겼고, 메인 화면 우측 상단에는 메모요와 더치페이 계산기와 동일한 응원 버튼을 얹었다. 메모 입력 칸은 실사용에 불필요해 제거했고, 복용 시간에 맞춰 약 이름 몇 알 드실 시간입니다 라는 푸시 알림이 한 번 오도록 알림 연동도 설계에 올렸다. 밤 늦게까지 재시작 크래시와 objective_c 에러 디버깅을 반복하다 초기 커밋 484981029 로 푸시를 걸었다. 이 저장소의 정식 시작점이 이 커밋이다.

고독사 방지앱은 오후에 새 프로젝트로 뼈대를 세웠다. fvm flutter create 로 lonely_death_prevention 프로젝트를 만들고 패키지명은 com.lonelydeath.prevention 으로 지정했다. 앱 이름은 고독사 방지앱 으로 iOS Info.plist 와 Android AndroidManifest 양쪽에 반영했다. 로고는 두 사람이 연결된 이미지에서 시작해 결국 두 손이 서로 잡고 있는 노란 원형 로고로 굳혔다. 노인도 한눈에 알아볼 수 있는 톤이 목표였고, iOS 아이콘은 처음부터 불투명 배경으로 잡아 앱스토어 리젝 여지를 없앴다. Material 3 테마에 고령자 친화 색상을 적용했고, flutter_native_splash 로 스플래시 배경을 #FFC107 로 다시 생성했다. 초반 빌드에서 text_style.dart 의 fontSize 관련 assertion 에러가 터져 해당 텍스트 스타일 설정을 정리하면서 안정화했다. 체크인 화면의 괜찮아요 버튼에는 메모요 프로젝트에서 가져온 고양이 아이콘을 붙여 분위기를 맞췄다. 오후 중반 즈음 앱 컨셉이 한 번 크게 바뀌었다. 사용자가 앱을 자주 열지 않아도 되는 자동 활동 판단 중심의 단순한 앱으로 방향을 틀고, CLAUDE.md 도 그 컨셉에 맞춰 전체 교체했다. 재시작 크래시도 메모요와 비슷한 원인이 있는지 확인하면서 GeneratedPluginRegistrant 관련 수정을 시도했고, 당일은 완전 안정화까지는 가지 못했다.

그 밖에 자잘한 작업과 상담도 있었다. 이 세션에서 누가 메인 에이전트고 서브에이전트인지, 오케스트레이터가 뭔지 같은 구조 질문이 오갔고, 클로드 데스크탑의 코드가 코워크 세션에서 약먹자를 건드리는 동안 현재 세션에서는 무엇을 해야 하는지를 조율했다. 용어 수준에서는 fvm 신규 빌드 명령, 릴리즈 빌드와 디버그 빌드의 차이, apk split 옵션의 동작 같은 것도 짧게 정리했다.
