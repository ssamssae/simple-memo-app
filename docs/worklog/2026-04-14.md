# 2026-04-14 작업일지

## 요약

메모요 앱의 UI를 애플 시스템 다크 투톤을 기반으로 전면 리디자인했다. 배경을 0xFF1C1C1E / 0xFF2C2C2E 두 톤으로 정리하고 모든 텍스트·아이콘을 amber 포인트 컬러로 통일했으며, 편집·응원·저장·뒤로 버튼을 하트 배지 스타일로 통일해 좌우 대칭으로 재배치했다. 기능 면에서는 좌상단 편집 진입 → 라디오 선택 → 일괄 삭제로 이어지는 복수 선택 삭제 플로우를 새로 추가했고, 스와이프/즐겨찾기/스크롤 safe area 관련 인터랙션 문제들을 함께 정리했다. 사용되지 않는 cupertino_icons 의존성을 제거해 번들을 가볍게 만들고, Android APK·Web(gh-pages)·iOS TestFlight 빌드까지 1.0.0+5 버전으로 배포했다. 마지막으로 매일 반복되는 작업일지 작성과 아이폰 릴리즈 실행 흐름을 Claude Code 커스텀 스킬(`/worklog`, `/irun`)로 자동화했다.

## UI / 디자인

- 다크 시스템 투톤 배경 적용 (0xFF1C1C1E / 0xFF2C2C2E)
- 모든 텍스트를 amber 포인트 컬러로 통일 (타이틀, 메모, 저장, 뒤로가기 아이콘, 드래그 핸들 등)
- AppBar 타이틀 붙여쓰기: "새메모" / "메모수정"
- 편집 · 응원 · 저장 · 뒤로 버튼을 하트 배지 형태로 통일, 좌우 20px 대칭 배치
- 메모 목록 14px 라운딩 카드 처리
- 스플래시 화면 및 iOS LaunchScreen 검은 배경으로 통일
- FAB 배경색을 메모 글씨색(amber)과 동일하게 맞춤

## 기능 추가

### 복수 선택 삭제
- 좌상단 "편집" 배지 → 선택 모드 진입
- 각 메모 왼쪽 라디오 점, 탭으로 선택 토글
- 상단 "삭제 (N)" 버튼 → 확인 다이얼로그("메모삭제") → 일괄 삭제

## 개선 / 버그 수정

- 스와이프 양방향 복구: 열린 상태에서 반대 방향 드래그 시 원위치
- 다른 메모 탭 시 열린 스와이프 먼저 닫히고 탭은 무시
- 즐겨찾기 해제 시 일반 그룹 최상단으로 이동
- 즐겨찾기된 메모 행의 별 아이콘 탭 → 해제 대신 메모 수정 진입
- 메모 목록/수정 화면 모두 홈 인디케이터 safe area 반영 (스크롤 시 가려짐 해결)
- 메모수정 빈 영역 드래그로도 스크롤 동작
