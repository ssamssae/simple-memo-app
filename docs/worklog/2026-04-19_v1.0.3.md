2026.04.19 작업일지 v1.0.3

v1.0.2 까지가 Tailscale rsync 복구와 Stop 훅 flush 재시도 루프로 아침과 점심의 인프라 정비를 마무리한 지점이었다면, v1.0.3 은 그 이후 오후부터 저녁까지의 흐름을 담는다. 크게 보면 할일 자동화 안정화, 더치페이 계산기 1.0.2+4 정식화, 데스크탑 night-runner 가 만든 morning-briefing 통합, 그리고 이번 세션에서의 APK 용량 최적화 네 축으로 정리된다.

오후 2시부터 3시 사이에는 할일 자동화가 한번 더 정리됐다. 오전부터 이어진 post-commit 훅의 중복 삽입 버그를 잡고, todos.md 쪽에서는 자동화 안정화 5번 항목을 완료로 처리하며 대규모 정리까지 한번에 밀어 넣었다. 같은 시간대에 daejong-page 쪽에서는 작업일지 산문체 뷰 페이지를 새로 만들어 피드 각 항목에 버튼을 달았고, 홈 소개 문구에는 Claude Max 5x에서 20x 로 올린 결제 이력까지 추가해 오늘자 기준으로 공개 홈페이지가 최신 상태로 맞춰졌다. 이어서 cancelled 타임라인의 TZ 버그, 즉 오늘(KST) 그룹이 누락되던 문제를 브라우저 new Date 의 toISOString 동작을 UTC 앵커로 바꿔 고정했다. 스킬 쪽에서는 memoyo-release 와 daejong-page-sync 커스텀 서브에이전트 두 개를 agents 디렉터리에 추가했고, /todo 스킬 문서에는 지우자 삭제 같은 표현도 하드삭제 대신 보류 취소 아카이브로 이동하라는 규칙을 명문화했다.

오후 4시 50분경부터는 더치페이 계산기 1.0.2+4 정식화에 집중했다. 먼저 F3 아이콘을 Pretendard Black 기반에 원화 코인과 네 코너를 넣은 안으로 리디자인했고, 홈 화면의 응원 버튼을 제거하면서 Flutter SDK 마이그레이션과 문구 튜닝을 한 번에 반영했다. daejong-page 포트폴리오와 타임라인에도 새 아이콘을 바로 적용했다. 뒤이어 Android 릴리스 서명 설정을 마치고 iOS 와 Android 를 1.0.1+3 빌드 준비 상태까지 올린 뒤, 번들 아이디를 com.daejongkang.dutch_pay_calculator 에서 com.daejongkang.dutchpay 로 변경해 스토어 식별자를 정돈했다. 마지막으로 개인정보처리방침 페이지를 홈페이지에 따로 추가해 스토어 심사 링크 걸이까지 완성했다.

저녁 6시 25분에는 윈도우 데스크탑에서 night-runner 기반 작업이 한 덩어리 도착했다. 처음 제안은 A 안과 B 안 두 갈래였는데 데스크탑이 실제로 밀어 올린 건 B 안, 즉 WSL 이 밤사이 janitor 브랜치로 PR 만 만들어 두고 맥 본진 morning-briefing 이 아침 7시 15분에 그 PR 을 모아 텔레그램에 리뷰 섹션으로 올리는 방식이다. 공유 스킬 저장소에 morning-briefing 1-E 섹션이 새로 들어왔고, 맥은 그걸 pull 로 받아 반영했다. 검토 결과 방향 자체는 맞지만 아직 PR 머지 답장 처리 로직이 없고 handoff-to-mac 스킬의 orphan 여부를 확인해야 하는 상태라, 이 부분들을 텔레그램으로 먼저 사용자에게 보고했다.

저녁 7시 반 직전부터는 오늘의 마지막 축인 더치페이 APK 용량 최적화를 돌렸다. 기본 릴리스 APK 가 41 메가로 나오던 이유가 armeabi v7a 와 arm64 v8a 와 x86_64 세 ABI 가 한 번에 묶여 있었기 때문임을 analyze size 결과로 확인했고, split per abi 옵션을 넣어 세 개를 분리했다. arm64 버전이 14 메가로 떨어져 전체 대비 63 퍼센트 감소했고, 바로 그 파일을 텔레그램으로 보냈다. 같은 흐름에서 /irun 으로 아이폰에는 clean + release 재설치를 돌려 2.3 초 install 로 마무리했다. Play Store 배포 때는 appbundle 로 올리면 구글이 기기별 자동 split 을 해 준다는 점을 메모로 남겨 두었다.
